Secure API Testing & Authorization Validation

## Objective
To understand REST APIs and perform basic security testing to validate authentication, authorization, input validation, and rate limiting.

## Concepts Covered
- REST API fundamentals
- API authentication testing

## Testing Performed
- Tested APIs with valid and invalid authentication tokens
- Checked unauthenticated access behavior
- Modified resource identifiers to test authorization
- Sent malformed inputs to test validation
- Observed API response codes and error messages
- Studied rate limiting behavior

## Identified Risks
- Broken Authentication
- Broken Authorization (BOLA)
- Improper Input Validation
- Lack of Rate Limiting
- Excessive Error Information

## Security Recommendations
- Enforce strong authentication
- Implement proper authorization checks
- Apply strict input validation
- Enable rate limiting
- Avoid sensitive error disclosures
